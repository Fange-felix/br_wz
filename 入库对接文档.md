## 二、文章状态与目录对应关系

| 文章状态 | 目录路径 | 说明 |
|---------|---------|------|
| 草稿 | `草稿/行业名/` | 文章撰写中，未审核 |
| 待入库 | `待入库/行业名/` | 审核通过，等待入库 |
| 已发布 | `已发布/行业名/` | 已入库并发布到网站 |
| 作废 | `作废/行业名/` | 弃用，不再处理 |

> ⚠️ 文章状态变更时，需手动移动文件到对应目录，并更新 `文章索引.csv`。

    "h1": "文章H1标题",

      {

  },
  "seo_checklist": {
    "title_length_chars": 0,

## 四、数据库表设计建议

### 4.1 文章主表（`articles`）

| 字段名 | 类型 | 说明 | JSON 对应路径 |
|--------|------|------|----------------|
| `id` | INT AUTO_INCREMENT | 主键 | — |
| `article_id` | VARCHAR(20) | 文章编号（如 BR-001） | `$.article_id` |
| `industry` | VARCHAR(50) | 所属行业 | `$.industry` |
| `status` | VARCHAR(20) | 文章状态 | `$.status` |
| `primary_keyword` | VARCHAR(100) | 主关键词 | `$.meta_info.primary_keyword` |
| `secondary_keywords` | JSON | 副关键词数组 | `$.meta_info.secondary_keywords` |
| `search_intent` | VARCHAR(50) | 搜索意图 | `$.meta_info.search_intent` |
| `title_tag` | VARCHAR(200) | SEO标题 | `$.meta_info.title_tag` |
| `meta_description` | VARCHAR(300) | Meta描述 | `$.meta_info.meta_description` |
| `url_slug` | VARCHAR(200) | URL别名 | `$.meta_info.url_slug` |
| `h1` | VARCHAR(300) | H1标题 | `$.content.h1` |
| `h2_sections` | JSON | H2章节（含正文） | `$.content.h2_sections` |
| `word_count` | INT | 文章字数（可计算） | — |
| `seo_checklist` | JSON | SEO检查项 | `$.seo_checklist` |
| `notes` | TEXT | 备注 | `$.notes` |
| `imported_to_db` | TINYINT(1) | 是否已入库 | `$.imported_to_db` |
| `created_at` | DATE | 创建日期 | `$.created_at` |
| `updated_at` | DATE | 更新日期 | `$.updated_at` |
| `github_path` | VARCHAR(300) | GitHub仓库相对路径 | — |

### 4.2 建表 SQL（MySQL）

```sql
CREATE TABLE `articles` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `article_id` VARCHAR(20) NOT NULL UNIQUE COMMENT '文章编号，如BR-001',
  `industry` VARCHAR(50) NOT NULL COMMENT '所属行业',
  `status` VARCHAR(20) NOT NULL DEFAULT '草稿' COMMENT '文章状态：草稿/待入库/已发布/作废',
  `primary_keyword` VARCHAR(100) NOT NULL COMMENT '主关键词',
  `secondary_keywords` JSON COMMENT '副关键词数组',
  `search_intent` VARCHAR(50) COMMENT '搜索意图',
  `title_tag` VARCHAR(200) COMMENT 'SEO标题',
  `meta_description` VARCHAR(300) COMMENT 'Meta描述',
  `url_slug` VARCHAR(200) COMMENT 'URL别名',
  `h1` VARCHAR(300) NOT NULL COMMENT 'H1标题',
  `h2_sections` JSON NOT NULL COMMENT 'H2章节数组，含正文',
  `word_count` INT DEFAULT 0 COMMENT '文章字数',
  `seo_checklist` JSON COMMENT 'SEO检查项',
  `notes` TEXT COMMENT '文章备注',
  `imported_to_db` TINYINT(1) DEFAULT 0 COMMENT '是否已入库：0否1是',
  `created_at` DATE COMMENT '创建日期',
  `updated_at` DATE COMMENT '更新日期',
  `github_path` VARCHAR(300) COMMENT 'GitHub仓库相对路径',
  `created_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  `updated_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_article_id` (`article_id`),
  INDEX `idx_industry` (`industry`),
  INDEX `idx_status` (`status`),
  INDEX `idx_primary_keyword` (`primary_keyword`),
  FULLTEXT INDEX `ft_h1` (`h1`),
  FULLTEXT INDEX `ft_h2_sections` (`h2_sections`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='SEO文章主表';
```

## 五、入库操作步骤

### 方式一：手动单条入库（适合调试）

1. 从 GitHub 仓库下载对应 JSON 文件
2. 读取 JSON 内容，映射到 `articles` 表字段
3. 执行 INSERT SQL
4. 将 `imported_to_db` 改为 `true`，`db_id` 填写数据库自增 ID
5. 更新 `文章索引.csv`，将对应行的「是否入库」改为「是」，「入库ID」填写数据库 ID
6. 将文件从 `草稿/` 移动到 `待入库/`（或 `已发布/`，视流程而定）
7. 提交并推送到 GitHub

### 方式二：批量入库脚本（推荐）

参考仓库根目录 `scripts/import_articles.py`（如不存在可联系小吴编写）。

## 六、文章索引 CSV 字段说明

文件位置：`文章索引.csv`

| 列名 | 说明 | 示例 |
|------|------|------|
| 文章编号 | 对应 JSON 中的 `article_id` | BR-001 |
| 主关键词 | 对应 `meta_info.primary_keyword` | 硅PU篮球场多少钱一平方 |
| 副关键词 | 用 `|` 分隔 | 硅PU篮球场价格\|硅PU篮球场造价明细 |
| 行业 | 对应 `industry` | 硅PU球场 |
| 文章状态 | 对应 `status` | 草稿 |
| 存放路径 | GitHub 相对路径 | 草稿/硅PU球场/BR-001-硅PU篮球场多少钱一平方.json |
| 是否入库 | 是/否 | 否 |
| 入库ID | 数据库中的 ID | （空） |
| 创建日期 | 对应 `created_at` | 2026-06-28 |
| 更新日期 | 对应 `updated_at` | 2026-06-28 |
| URL_Slug | 对应 `meta_info.url_slug` | gui-pu-lan-qiu-chang-... |

> 💡 每次入库完成后，需同步更新此 CSV 文件并推送到 GitHub，保持索引与数据库一致。

## 七、注意事项

1. **文件移动即状态变更**：文章状态变更请移动文件到对应目录，不要只改 JSON 里的 `status` 字段
2. **更新日期**：文章有修改时，需同步更新 JSON 中的 `updated_at` 字段
3. **入库标记**：入库后务必将 `imported_to_db` 改为 `true`，避免重复入库
4. **编码格式**：所有 JSON 文件均为 UTF-8 编码，CSV 文件为 UTF-8 with BOM（Excel 可直接打开）
5. **文章模板**：新建文章时复制 `文章模板.json`，填写对应字段，放入 `草稿/对应行业/` 目录
