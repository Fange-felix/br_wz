### 目录结构

```
br_wz/
├── 草稿/硅PU球场/        ← 新文章，尚未通知入库
├── 待入库/硅PU球场/      ← 已通知，可以入库
├── 已发布/硅PU球场/      ← 已入库，发布上线
├── 作废/硅PU球场/        ← 废弃，不处理
├── scripts/               ← 入库脚本
├── 文章模板.json           ← 新建文章时的结构模板
├── 文章索引.csv           ← 所有文章汇总表
└── 入库技术文档.md        ← 本文档
```

**入库时只处理 `待入库/硅PU球场/` 目录下的文件。**

## 二、帝国CMS栏目ID

| ID | 栏目名 | 用途 |
|----|--------|------|
| **14** | 公司动态 | 公司新闻、案例、活动等 |
| **15** | 行业动态 | 行业资讯、市场分析等 |

每篇文章 JSON 中通过 `classid` 字段指定栏目，入库脚本根据此值自动分栏。

## 四、JSON → 帝国CMS字段映射表

| 帝国CMS字段 | JSON 路径 | 类型 | 说明 |
|-------------|-----------|------|------|
| `classid` | `$.classid` | INT | 栏目ID（14=公司动态 / 15=行业动态） |
| `title` | `$.content.h1` | VARCHAR | 文章标题（对应帝国CMS title字段） |
| `titlepic` | `$.images.title_pic` | VARCHAR | 封面图URL |
| `newstime` | `$.created_at` | DATETIME | 发布时间 |
| `keyboard` | `$.meta_info.primary_keyword` + `$.meta_info.secondary_keywords` | VARCHAR | 关键词（逗号分隔） |
| `smalltext` | `$.meta_info.meta_description` | VARCHAR | 简介摘要 |
| `newstext` | `$.content.html_content` | LONGTEXT | 正文HTML（直接可用，无需拼接） |
| `writer` | — | VARCHAR | 作者（可固定值或留空） |
| `userid` | — | INT | 发布者用户ID |
| `checked` | — | TINYINT | 审核状态（1=已审核） |
| `istop` | — | TINYINT | 是否置顶（0=否） |
| `isgood` | — | TINYINT | 是否推荐（0=否） |

### 自定义字段映射（如有扩展）

| 自定义字段 | JSON 路径 | 说明 |
|-----------|-----------|------|
| `seo_title` | `$.meta_info.title_tag` | SEO Title标签 |
| `seo_description` | `$.meta_info.meta_description` | SEO Meta描述 |
| `url_slug` | `$.meta_info.url_slug` | URL别名 |
| `word_count` | `$.content.word_count` | 文章字数 |
| `article_id` | `$.article_id` | 文章编号（BR-XXX） |
| `content_images` | `$.images.content_images` | 正文图片（JSON数组） |

## 五、数据库建表SQL（MySQL）

```sql
CREATE TABLE `seo_articles` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `article_id` VARCHAR(20) NOT NULL UNIQUE COMMENT '文章编号 BR-XXX',
  `industry` VARCHAR(50) NOT NULL COMMENT '所属行业',
  `classid` INT NOT NULL DEFAULT 15 COMMENT '帝国CMS栏目ID（14=公司动态/15=行业动态）',
  `status` VARCHAR(20) NOT NULL DEFAULT '草稿' COMMENT '状态',
  `primary_keyword` VARCHAR(100) NOT NULL COMMENT '主关键词',
  `secondary_keywords` JSON COMMENT '副关键词',
  `search_intent` VARCHAR(50) COMMENT '搜索意图',
  `title_tag` VARCHAR(200) COMMENT 'SEO Title标签',
  `meta_description` VARCHAR(300) COMMENT 'Meta描述',
  `url_slug` VARCHAR(200) COMMENT 'URL别名',
  `h1` TEXT COMMENT 'H1标题',
  `word_count` INT DEFAULT 0 COMMENT '文章字数',
  `html_content` LONGTEXT COMMENT '正文HTML（预渲染）',
  `h2_sections` JSON COMMENT 'H2章节数组（结构化数据）',
  `title_pic` VARCHAR(500) COMMENT '封面图URL',
  `content_images` JSON COMMENT '正文图片URL数组',
  `imported_to_db` TINYINT(1) DEFAULT 0 COMMENT '是否已入库',
  `created_at` DATE COMMENT '创建日期',
  `updated_at` DATE COMMENT '更新日期',
  `file_path` VARCHAR(300) COMMENT 'GitHub相对路径',
  INDEX `idx_classid` (`classid`),
  INDEX `idx_industry` (`industry`),
  INDEX `idx_status` (`status`),
  INDEX `idx_primary_keyword` (`primary_keyword`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='SEO文章表';
```

```php
  `classid`, `title`, `titlepic`, `newstime`, `keyboard`,

  1,                                      -- checked: 已审核
  0,                                      -- istop: 不置顶
  0,                                      -- isgood: 不推荐
  1                                       -- userid: 发布者ID
);
```

## 八、文章索引CSV字段说明

文件：`文章索引.csv`

| 列名 | 说明 | 示例 |
|------|------|------|
| 文章编号 | 唯一编号 | BR-001 |
| 主关键词 | SEO主词 | 硅PU篮球场多少钱一平方 |
| 副关键词 | `|` 分隔 | 硅PU篮球场价格\|硅PU篮球场造价明细 |
| 行业 | 所属行业 | 硅PU球场 |
| 栏目ID | 帝国CMS栏目ID | 15 |
| 文章状态 | 草稿/待入库/已发布/作废 | 草稿 |
| 存放路径 | 相对路径 | 草稿/硅PU球场/BR-001-...json |
| 是否入库 | 是/否 | 否 |
| 入库ID | 数据库ID | （空） |
| 创建日期 | YYYY-MM-DD | 2026-06-28 |
| 更新日期 | YYYY-MM-DD | 2026-06-28 |
| URL_Slug | URL别名 | gui-pu-lan-qiu-chang-... |

> 入库后更新「是否入库」为「是」，「入库ID」填写数据库ID。

## 九、注意事项

1. **编码**：所有文件 UTF-8，CSV 为 UTF-8 with BOM（Excel 兼容）
2. **html_content**：预渲染好的完整HTML正文，入库时直接取用，无需拼接
3. **classid**：只有 14（公司动态）和 15（行业动态）两个值
4. **文章移动**：入库完成后，由小吴将文件从 `待入库/` 移到 `已发布/`
5. **索引同步**：入库后告知小吴更新 `文章索引.csv`
6. **编号唯一**：`BR-XXX` 由小吴统一分配
