## 三、文章 JSON 结构（完整字段说明）

每篇文章为一个 JSON 文件，编码：**UTF-8**。

### 顶层结构

```json
{
  "article_id": "BR-001",
  "industry": "硅PU球场",
  "status": "草稿",
  "created_at": "2026-06-28",
  "updated_at": "2026-06-28",
  "imported_to_db": false,
  "db_id": null,
  "meta_info": { ... },
  "content": { ... },
  "seo_checklist": { ... }
}
```

### `meta_info` 字段

| 字段 | 类型 | 说明 | 对应数据库字段 |
|------|------|------|---------------|
| `primary_keyword` | string | 主关键词 | `primary_keyword` |
| `secondary_keywords` | array | 副关键词列表 | `secondary_keywords` (JSON) |
| `title_tag` | string | 页面 Title 标签（≤30字符） | `title_tag` |
| `meta_description` | string | Meta 描述（≤78字符） | `meta_description` |
| `url_slug` | string | URL 别名 | `url_slug` |

### `content` 字段

| 字段 | 类型 | 说明 | 对应数据库字段 |
|------|------|------|---------------|
| `h1` | string | 文章 H1 标题 | `h1` |
| `h2_sections` | array | H2 章节数组，每项为 `{ "title": "...", "content": "..." }` | `content` (JSON 或长文本) |
| `word_count` | int | 文章总字数 | `word_count` |

### `seo_checklist` 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `keyword_in_title` | bool | 关键词是否出现在 Title |
| `keyword_in_h1` | bool | 关键词是否出现在 H1 |
| `keyword_density_ok` | bool | 关键词密度是否 2%-5% |
| `has_internal_links` | bool | 是否有内链 |
| `has_faq_schema` | bool | 是否有 FAQ 结构化数据 |

## 四、数据库表结构（MySQL）

```sql
CREATE TABLE `seo_articles` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `article_id` VARCHAR(20) NOT NULL UNIQUE COMMENT '文章编号 BR-XXX',
  `industry` VARCHAR(50) NOT NULL COMMENT '所属行业',
  `status` VARCHAR(20) NOT NULL DEFAULT '草稿' COMMENT '状态',
  `primary_keyword` VARCHAR(100) NOT NULL COMMENT '主关键词',
  `secondary_keywords` JSON COMMENT '副关键词',
  `title_tag` VARCHAR(200) COMMENT '页面Title',
  `meta_description` VARCHAR(300) COMMENT 'Meta描述',
  `url_slug` VARCHAR(200) COMMENT 'URL别名',
  `h1` TEXT COMMENT 'H1标题',
  `content` LONGTEXT COMMENT '正文JSON或HTML',
  `word_count` INT DEFAULT 0 COMMENT '字数',
  `imported_to_db` TINYINT(1) DEFAULT 0 COMMENT '是否已入库',
  `created_at` DATE COMMENT '创建日期',
  `updated_at` DATE COMMENT '更新日期',
  `file_path` VARCHAR(300) COMMENT 'GitHub相对路径',
  INDEX `idx_industry` (`industry`),
  INDEX `idx_status` (`status`),
  INDEX `idx_primary_keyword` (`primary_keyword`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='SEO文章表';
```

## 五、入库 SQL 示例

### 单条入库

```sql
INSERT INTO `seo_articles` (
  `article_id`, `industry`, `status`, `primary_keyword`,
  `secondary_keywords`, `title_tag`, `meta_description`, `url_slug`,
  `h1`, `content`, `word_count`, `imported_to_db`,
  `created_at`, `updated_at`, `file_path`
) VALUES (
  'BR-001',
  '硅PU球场',
  '已发布',
  '硅PU篮球场多少钱一平方',
  '["硅pu篮球场造价","硅pu篮球场价格","硅pu篮球场施工报价"]',
  '硅PU篮球场多少钱一平方（2026最新价格）',
  '2026年硅PU篮球场造价明细，含3mm/4mm/5mm厚度报价，包工包料价格参考。',
  '/xinpucaiqiu/duoshaoqian',
  '硅PU篮球场多少钱一平方（2026年最新价格参考）',
  '{"h2_sections": [...]}',  -- 完整JSON或转HTML后存入
  1280,
  1,
  '2026-06-28',
  '2026-06-28',
  '已发布/硅PU球场/BR-001-硅PU篮球场多少钱一平方.json'
);
```

### 批量入库（Python 脚本）

```python
import json
import mysql.connector
import os

# 读取待入库目录
pending_dir = '待入库/硅PU球场'
files = [f for f in os.listdir(pending_dir) if f.endswith('.json')]

conn = mysql.connector.connect(
    host='localhost', user='root', password='password',
    database='seo_db', charset='utf8mb4'
)
cursor = conn.cursor()

for filename in files:
    with open(os.path.join(pending_dir, filename), 'r', encoding='utf-8') as f:
        article = json.load(f)

    sql = """
    INSERT INTO seo_articles
    (article_id, industry, status, primary_keyword, secondary_keywords,
     title_tag, meta_description, url_slug, h1, content, word_count,
     imported_to_db, created_at, updated_at, file_path)
    VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s)
    """

    h2_content = json.dumps(article['content']['h2_sections'], ensure_ascii=False)

    cursor.execute(sql, (
        article['article_id'],
        article['industry'],
        article['status'],
        article['meta_info']['primary_keyword'],
        json.dumps(article['meta_info']['secondary_keywords'], ensure_ascii=False),
        article['meta_info']['title_tag'],
        article['meta_info']['meta_description'],
        article['meta_info']['url_slug'],
        article['content']['h1'],
        h2_content,
        article['content']['word_count'],
        1 if article['imported_to_db'] else 0,
        article['created_at'],
        article['updated_at'],
        f"待入库/硅PU球场/{filename}"
    ))

conn.commit()
cursor.close()
conn.close()
```

## 七、注意事项

1. **编码**：所有文件 UTF-8，数据库表 `utf8mb4`
2. **content 字段存储**：可选择存原始 JSON，或渲染为 HTML 后存储
3. **文章移动**：入库完成后，由小吴将文件从 `待入库/` 移到 `已发布/`
4. **索引同步**：入库后务必告知小吴更新 `文章索引.csv` 的 `是否入库` 和 `入库ID`
5. **编号唯一**：`BR-XXX` 由小吴统一分配，不要自行修改
