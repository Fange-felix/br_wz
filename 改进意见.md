# br_wz 仓库改进意见

小海（AI助手）审阅了 `br_wz` 文章管理仓库的整体设计，提出以下改进建议：

---

## 一、JSON 结构优化

### 1.1 content 字段建议直接包含 HTML 正文

当前结构：
```json
{
  "content": {
    "h1": "标题",
    "h2_sections": [
      {"h2": "一、章节", "content": "正文"}
    ]
  }
}
```

**问题：** 入库时需要把 h2_sections 拼成 HTML，多了一步转换。

**建议：** 增加一个 `html_content` 字段，预先渲染好完整 HTML 正文：
```json
{
  "content": {
    "h1": "标题",
    "h2_sections": [...],
    "html_content": "<h2>一、章节</h2><p>正文</p>..."
  }
}
```
这样入库脚本可以直接取 `html_content`，不需要自己拼接。

### 1.2 建议增加 `classid` 字段

目前 JSON 里没有指定帝国CMS栏目ID，入库时无法自动分栏。建议增加：

```json
{
  "classid": 14,
  "industry": "硅PU球场"
}
```

**栏目ID只限以下两个：**
| ID | 栏目名 | 用途 |
|----|--------|------|
| **14** | 公司动态 | 公司新闻、案例、活动等 |
| **15** | 行业动态 | 行业资讯、市场分析等 |

### 2.1 增加「栏目ID」列
```csv
文章编号,主关键词,行业,栏目ID,文章状态,存放路径,是否入库,入库ID,URL_Slug
```

### 2.2 CSV 编码
目前是 UTF-8 with BOM，Excel 可以正常打开，这个保持就好。

---

## 三、工作流建议

### 3.1 文件状态自动化
建议在仓库根目录加一个 `scripts/` 目录，存放：
- `sync_from_github.php` — 从 GitHub 拉取文章并入库到帝国CMS（公司动态 classid=14 / 行业动态 classid=15）
- `move_to_published.php` — 入库后自动移动文件 + 更新索引

### 3.2 草稿→待入库→已发布 流程
当前流程需要手动移动文件，建议添加自动化脚本配合 GitHub Actions：
- 当文件移入 `待入库/` 时，自动触发入库脚本
- 入库成功后自动移入 `已发布/` 并更新索引

---

## 四、文章模板改进

### 4.1 word_count 缺少
当前模板没有 `word_count` 字段，但在入库时帝国CMS需要字数统计。建议增加：
```json
{
  "word_count": 0,
  "content": {
    ...
    "word_count": 0
  }
}
```

### 4.2 图片处理
目前 JSON 只含文本，建议增加图片字段：
```json
{
  "images": {
    "title_pic": "封面图URL",
    "content_images": ["正文图1URL", "正文图2URL"]
  }
}
```

---

## 五、文档更新

### 5.1 `入库对接文档.md` 和 `同事对接说明.md` 内容重复较多
两篇文档功能有重叠，建议合并为一篇，另一篇作为简短索引。

### 5.2 补充字段映射表
建议在文档中增加 JSON → 帝国CMS数据库字段的完整映射表，方便入库同事直接对照。
